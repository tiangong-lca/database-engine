# Remote Schema Workspace

This directory stores the latest remote schema dump and the split-out workspace generated from the `dev` database.

## Generated Content

The following paths are generated by `python scripts/build_schema_workspace.py --environment dev`:

- `remote_schema.sql`
- `global/`
- `schemas/`

## Refresh Behavior

Each refresh performs these operations inside `supabase/workspace`:

- `remote_schema.sql` is overwritten with the latest exported dump.
- `global/` is deleted and recreated from the latest dump.
- `schemas/` is deleted and recreated from the latest dump.
- `README.md` is preserved.
- `changes/` is preserved and intended for manual edits that should survive refresh.
- Other root-level files are currently preserved, but only `README.md`, documentation files, and `changes/` should be treated as stable manual-edit locations.

## Important Warnings

- Any manual edits inside `remote_schema.sql`, `global/`, or `schemas/` will be lost the next time the workspace is refreshed.
- Any uncommitted Git changes inside `remote_schema.sql`, `global/`, or `schemas/` can be overwritten or removed during refresh.
- Review `git status` and commit or stash important work before running the refresh command.
- Treat this workspace as a generated view of the remote database, not as the source of truth for hand-maintained schema changes.

## Recommended Use

- Use `remote_schema.sql` for full raw-dump inspection.
- Use `global/` and `schemas/` for human-readable browsing and review.
- Copy objects you plan to modify into `changes/`, keeping the same relative structure as `schemas/`.
- Generate migrations from files under `changes/` or `supabase/model/schemas/` only when the source path matches one of the currently supported object types:
  - `functions/<name>/definition.sql`
  - `views/<name>/definition.sql`
  - `materialized_views/<name>/definition.sql`
  - `tables/<table>/policies/<name>.sql`
  - `tables/<table>/triggers/<name>.sql`
- Table definitions, indexes, sequences, schema files, and other workspace files are not currently supported as direct `new_migration.py` sources.
- Put permanent notes or operating instructions in `README.md` or other root-level documentation files, not inside generated folders.

## Refresh Command

```bash
python scripts/build_schema_workspace.py --environment dev
```
